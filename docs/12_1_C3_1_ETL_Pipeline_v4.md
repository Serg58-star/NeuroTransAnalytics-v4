# C3.1 — ETL-пайплайн для модели данных v4

## 1. Назначение документа

Настоящий документ описывает **ETL-пайплайн (Extract–Transform–Load)**
в архитектуре NeuroTransAnalytics версии v4.

В версии v4 ETL-пайплайн:

- **не занимается миграцией legacy-данных**;
- **не извлекает данные из внешних файлов**;
- работает **исключительно поверх базы данных `neuro_data.db`**.

Документ фиксирует **рабочий и окончательный статус ETL**,
а не проектное или переходное решение.

---

## 2. Методологическая позиция ETL в v4

ETL-пайплайн v4 предназначен для:

- извлечения данных из канонического хранилища;
- нормализации и проверки целостности;
- формирования событийного слоя для дальнейших вычислений (C3.2–C3.4).

ETL **не выполняет**:
- аналитических расчётов;
- статистической обработки;
- агрегации;
- интерпретации;
- визуализации.

ETL является **инженерным, а не исследовательским слоем**.
