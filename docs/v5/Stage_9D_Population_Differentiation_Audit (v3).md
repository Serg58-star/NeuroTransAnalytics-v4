# Stage 9D.0 — SQLite Schema Audit (Mandatory)

## Цель

Определить реальную структуру базы neuro_data.db
без предположений о названиях таблиц и полей.

---

## 9D.0.1 Обязательные действия

1. Получить список таблиц:
   SELECT name FROM sqlite_master WHERE type='table';

2. Для каждой таблицы:
   PRAGMA table_info(table_name);

3. Определить:

   - Таблицу(ы), содержащие идентификатор субъекта
   - Таблицу(ы), содержащие данные тестирования
   - Таблицу(ы), содержащие демографические данные
   - Наличие поля пола (и его кодировку)
   - Наличие данных, позволяющих вычислить возраст
   - Наличие даты тестирования
   - Наличие нескольких тестов на одного субъекта

4. Построить схему связей таблиц (join-логика).

---

## 9D.0.2 Deliverables

1. SQLite_Schema_Audit_Report.md
2. Диаграмма связей таблиц
3. Список ключевых полей
4. Явное описание:
   - как кодируется пол
   - как хранится дата
   - как может быть вычислен возраст (если возможно)

---

## 9D.0.3 Запрещено

- предполагать названия полей
- выполнять статистический анализ
- интерпретировать данные
- применять критерии PASS/FAIL

Если возраст невозможно вычислить — аудит останавливается.

# Stage 9D.1 — Data Preparation Protocol

Выполняется только после завершения 9D.0.

---

## 9D.1.1 Вычисление возраста (если возможно)

Возраст вычисляется только на основании
реально обнаруженных полей.

Формула определяется на основании структуры базы,
зафиксированной в отчёте 9D.0.

Возраст рассчитывается на уровне теста,
если структура базы это позволяет.

---

## 9D.1.2 Исключение записей

Критерии исключения:

- Отсутствие демографических данных
- Отсутствие даты тестирования
- Невозможность вычислить возраст

Все исключения фиксируются количественно.

---

## 9D.1.3 Множественные тестирования

Если субъект имеет несколько тестов:

- каждая запись анализируется отдельно,
- при регрессии учитывать кластеризацию по субъекту
  (cluster-robust SE или mixed-effects).

Игнорирование кластеризации запрещено.

# Stage 9D.2 — Structural Differentiation Analysis

Выполняется только после завершения 9D.1.

Анализ проводится строго в:

- Z-space (robust standardized)
- Severity(Z)
- Covariance Σ
- Participation Ratio
- Effective Rank
- PC1 explained variance
- Silhouette

---

## I. Проверка различий по полу (Sex)

### 1. Различие центров

Δ_sex = || μ_group1 − μ_group2 ||₂

PASS: Δ ≤ 0.5 Z
FAIL: Δ > 1.0 Z

0.5–1.0 → Borderline

---

### 2. Различие ковариаций

PASS:
относительное расхождение < 10%

FAIL:
расхождение > 20%

---

### 3. Severity distribution difference

PASS: |Cohen’s d| < 0.3
FAIL: |d| ≥ 0.5

---

### 4. Silhouette (разметка по полу)

PASS: Silhouette < 0.15
FAIL: Silhouette ≥ 0.25

---

## II. Проверка возрастного градиента (Age)

### 1. Линейная зависимость Severity ~ Age

PASS: |ρ| < 0.2
FAIL: |ρ| ≥ 0.35

---

### 2. Регрессия (clustered)

PASS: R² < 0.05
FAIL: R² ≥ 0.10

---

### 3. Нелинейность

PASS:
нелинейная модель объясняет < 10% дисперсии

FAIL:
≥ 15%

---

### 4. Геометрический дрейф по возрастным квартилям

PASS:
изменения PR, PC1, центров ≤ 10%

FAIL:
> 20%

---

## III. Interaction (Sex × Age)

PASS:
interaction p ≥ 0.05

FAIL:
p < 0.01

---

# IV. Глобальный критерий Stage 9D

PASS:

FAIL:

ровно 1 FAIL
