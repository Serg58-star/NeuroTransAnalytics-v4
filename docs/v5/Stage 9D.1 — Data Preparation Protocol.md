# Stage 9D.1 — Data Preparation Protocol

## Исполнительная задача для GoAn

## Статус

ОБЯЗАТЕЛЬНО К ИСПОЛНЕНИЮ
Предусловие для перехода к Stage 9D.2

---

# 1. Цель этапа

Подготовить структурно корректный аналитический датасет из базы `neuro_data.db`
для последующего выполнения Stage 9D.2 — Structural Differentiation Analysis.

Этап 9D.1 является исключительно этапом подготовки данных.

На данном этапе запрещено:

* выполнять статистический анализ
* рассчитывать PASS/FAIL
* интерпретировать различия
* вычислять корреляции, регрессии, эффекты
* выполнять кластеризацию

---

# 2. Архитектурные инварианты (Нарушение запрещено)

Core v5 — LOCKED.

Изменения запрещены для следующих параметров:

* Z-space — без модификаций
* Формула Severity:

```
S(Z) = Z^T Σ^-1 Z
```

* κ = 0.08 — без изменений
* Anchored thresholds — зафиксированы на baseline (κ = 0)
* Центроид не вычитается
* Матрица Σ не пересчитывается

Stage 9D.1 не должен вмешиваться в геометрию пространства.

# 4. Обязательные операции

## 4.1 Формирование JOIN

Использовать только явный JOIN:

```sql
SELECT
    t.trial_id,
    t.subject_id,
    t.test_date,
    t.session_condition,
    u.birth_date,
    u.gender,
    t.*
FROM trials t
JOIN users u
ON t.subject_id = u.subject_id;
```

Запрещено:

* неявные соединения
* использование недокументированных полей

---

## 4.2 Вычисление возраста (на уровне каждого теста)

Возраст рассчитывается для каждой строки `trial`.

Формула:

```
Age = (test_date − birth_date) / 365.25
```

Требования:

* Явный парсинг дат
* Тип данных — FLOAT
* Округление запрещено
* Возраст сохраняется в каждой строке

Если дата не парсится → запись исключается.

---

## 4.3 Критерии исключения (строгие)

Исключить запись можно только если:

* `gender IS NULL`
* `birth_date IS NULL`
* `test_date IS NULL`
* ошибка парсинга даты
* отсутствует соответствие subject_id
* выявлена техническая порча записи

Запрещено исключать:

* экстремальные RT
* преждевременные нажатия
* пропуски
* длинные хвосты
* высокую вариабельность
* физиологические экстремумы

Запрещено:

* trimming
* winsorizing
* нормализация распределения

---

## 4.4 Множественные тестирования

Если субъект имеет несколько тестов:

* сохранить все строки
* не агрегировать
* не усреднять
* не объединять по субъекту

Каждый `trial` остаётся независимой строкой.

Кластеризация будет учитываться только на этапе 9D.2.

   "Статистический анализ на этапе Stage 9D.1 не выполнялся."

# 7. Критерий завершения

Stage 9D.1 считается завершённым только если:

* возраст вычислен для всех валидных строк
* лог исключений полностью задокументирован
* датасет воспроизводим из SQL
* инварианты Core v5 не нарушены

После выполнения — ожидать разрешения на запуск Stage 9D.2.
