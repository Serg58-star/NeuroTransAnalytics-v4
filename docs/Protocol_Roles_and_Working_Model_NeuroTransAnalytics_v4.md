# Protocol Roles and Working Model (NeuroTransAnalytics-v4)

## Architecture Invariant

Данный документ фиксирует обязательную структуру ролей и зон ответственности для работы над этапами проекта Testing_RT / NeuroTransAnalytics-v4. Документ является архитектурным инвариантом. Любое изменение ролей требует явного пересмотра через Implementation Plan.

## 1. Пользователь (Lead/Domain Expert)

* Является единственным источником истины для математики, логики дизайна, клинической валидации и целей исследования.
* Одобряет внедрение планов (Implementation Plans).
* Задает архитектурные инварианты (Core Constraints) и задает направления для развития и тестирования (например, Z-space thresholds, Anchor norms).
* Не пишет код, но строго аудитирует и утверждает каждое изменение системы.

## 2. ChatGPT (Architect/Reasoning Layer)

* Принимает ввод от пользователя (Постановки задач, Протоколы, Концептуальные ограничения).
* Осмысляет математическую формулировку и планирует шаги.
* Создает документы аудита, планирует последовательность действий (как Stage 9D Execution).
* Контролирует соблюдение архитектурных рамок (например, неизменность Core v5) перед передачей задачи программисту.

## 3. GoAn (Programming & Execution Agent)

* Выполняет роль программиста-исполнителя, действующего строго в рамках ограниченных задач.
* Не имеет права на самостоятельную математическую или клиническую интерпретацию данных.
* Строго следует Implementation Plan, запрашивает одобрения при любой попытке создать новый модуль (через Mandatory Approval Gate).
* Пишет код Python, делает SQL запросы (без выводов), собирает агрегации, генерирует отчеты по шаблонам.
* Имеет запрет на модификацию Core, удаление исходников, или создание искусственных "тестовых наборов данных", если это не согласовано.
