# Amendment to C3 Phase Completion Summary — v4.0.1

## 1. Основание

В рамках реализации сценариев пространственной симметрии (A0.3 и последующих A0.x) принято решение о контролируемом расширении слоя C3.3.

Это изменение не является пересмотром архитектуры, а представляет собой формализованное расширение агрегированного контракта.

---

## 2. Изменение контракта C3.3

### Было (v4.0.0):

AggregatedFrame агрегировался по:

```
subject_id, session_id, test_type
```

### Стало (v4.0.1):

AggregatedFrame агрегируется по:

```
subject_id, session_id, test_type, stimulus_location
```

---

## 3. Что НЕ изменилось

* ETL (C3.1) — без изменений;
* Component Timing (C3.2) — без изменений;
* QC-политика — без изменений;
* Формулы Median, MAD, IQR — без изменений;
* ΔV1 определение — без изменений;
* Derived layer формат — Parquet (fastparquet);
* GUI не выполняет вычислений.

---

## 4. Причина расширения

Поддержка пространственных сценариев (A0.3 и далее) требует сохранения spatial dimension на уровне агрегирования.

Без этого пространственная структура терялась на уровне C3.3.

---

## 5. Архитектурный статус C3

C3 переходит из состояния:

> Stable (v4.0.0)

в состояние:

> Extended (v4.0.1)

Расширение не нарушает воспроизводимость и не меняет вычислительную логику.

---

## 6. Обязательные действия

1. Регенерация всех derived-артефактов;
2. Проверка детерминированности;
3. Повторная проверка A0.0–A0.2 на корректность;
4. Обновление version registry при необходимости.

---

Статус: Amendment Applied.
