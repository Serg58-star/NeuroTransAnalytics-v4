# 1. Цель поправки

Усилить формулировки и обязательные требования Implementation Plan (Task 47 v2), чтобы:

1. Зафиксировать проверку соответствия аналитики Stage 1–9C реальной архитектуре теста.
2. Обязать реконструкцию исходной нейрофизиологической гипотезы.
3. Ввести количественную оценку реализации сценариев A/B/C.
4. Расширить обязательную матрицу параметров.
5. Строго регламентировать раздел Exploratory.
6. Добавить итоговый раздел об оценке степени изученности базы и потенциала дальнейшего анализа.

---

# 2. Обязательные изменения к Implementation Plan

## 2.1 Усиление Goal Description

Добавить к текущей формулировке цели:

> The engineering reconstruction must explicitly verify the correspondence between the real historical test protocol and the analytics performed in Stage 1–9C, identifying any structural deviations, omissions, or abstractions introduced by the modeling layers.

Обязательный результат:
- выявление точек расхождения модели и протокола;
- фиксация неиспользованных параметров теста.

---

## 2.2 Обязательная реконструкция исходной гипотезы

Добавить подраздел:

### Reconstruction of the Original Neurophysiological Hypothesis

Требуется:

- восстановить исходную гипотезу декомпозиции (∆V1, ∆V4, ∆V5/MT);
- разделить:
  - подтверждённые элементы;
  - непроверенные элементы;
  - декларативные предположения;
- указать, какие элементы гипотезы реально тестировались в Stage 1–9C.

---

## 2.3 Количественная оценка реализации сценариев

В разделе аудита A/B/C добавить обязательные требования:

1. Для каждого сценария указать:
   - статус (реализован / частично / не реализован);
   - степень реализации в % (оценочно, с обоснованием);
   - наличие отчёта или вычислительного результата.
2. Сформировать таблицу:

| Сценарий | Уровень | Статус | % реализации | Наличие отчёта | Приоритет |

3. Провести ранжирование нереализованных сценариев по потенциальной значимости.

---

## 2.4 Расширение матрицы параметров

Матрица «Параметр теста → Аналитика → v5» обязана включать:

- Поле зрения
- Цвет
- PSI
- Позиция в серии (1–36)
- Тройки (masking clusters)
- Смещение (0/1/2/3)
- Предтестовые попытки (NoUchtPOKAZ_COUNT)
- Повторные попытки (anti-CV logic)
- RANO/POZDNO
- SrKvadrOtkl
- VidSost_txt
- Время суток
- Системные параметры (MinRedLight, MaxRedLight, ROTATE_PERIOD)

Для каждого параметра указать:
- использован ли;
- как использован;
- не использован;
- потенциал для v5.

---

## 2.5 Жёсткая регламентация Exploratory раздела

Добавить требования:

1. Разрешено:
   - любые статистические методы;
   - любые комбинации параметров;
   - моделирование альтернативных декомпозиций.
2. Обязательно:
   - математическое обоснование метода;
   - указание ограничений;
   - указание потенциального эффекта на v5.
3. Запрещено:
   - изменение архитектуры теста;
   - произвольное переопределение переменных без документированной причины.

---

## 2.6 Новый обязательный раздел

Добавить раздел:

### 7. Степень изученности базы neuro_data.db и оценка потенциала дальнейшего анализа

Требуется:

1. Оценить, исчерпан ли потенциал исторических данных.
2. Обосновать:
   - какие пласты ещё не вскрыты;
   - какие методы не применялись;
   - какие сценарии не реализованы;
   - есть ли diminishing returns (с аргументацией).
3. Не использовать декларативных утверждений без анализа.
