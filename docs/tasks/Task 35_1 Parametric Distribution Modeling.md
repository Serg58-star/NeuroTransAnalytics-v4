# Task 35.1 — Parametric Distribution Modeling (углубление Stage 6)

## Контекст

Task 35.1 является расширением Stage 6 (Variability & Tail Geometry) и не вводит нового этапа Framework.

Stage 6 установил:

* Распределения RT не являются гауссовыми.
* Наблюдаются выраженные heavy-tail эффекты.
* MAD встроен в Speed.
* Shape-показатели (skewness, tail ratios) частично ортогональны центральной тенденции.

Однако остаётся фундаментальный вопрос:

> Какой генеративный закон наиболее адекватно описывает распределение RT?

Task 35.1 направлен на параметрическое моделирование распределений RT
без изменения архитектуры данных и без мутации SQLite.

---

# Методологические ограничения

1. Используются trial-level RT.
2. Никакие данные не модифицируются.
3. Не вводятся новые агрегаты, только параметрические оценки.
4. Сравнение моделей проводится строго через критерии качества аппроксимации.
5. Интерпретации формулируются операционально.

---

# Цели

1. Определить, какое семейство распределений лучше описывает RT.
2. Проверить, доминирует ли одна модель популяционно.
3. Оценить, существуют ли подтипы распределений.
4. Связать параметры моделей с trait/state компонентами.

---

# Анализируемые семейства распределений

Для каждой 36-попыточной последовательности выполнить fit следующих моделей:

1. Normal
2. Lognormal
3. Gamma
4. Ex-Gaussian
5. Weibull

---

# Метрики сравнения

Для каждой модели рассчитываются:

* Log-likelihood
* AIC
* BIC
* KS-distance
* Anderson–Darling statistic

Дополнительно:

* QQ-plot deviation index
* Stability of parameter estimates (bootstrap)

---

# Популяционный анализ

1. Определить, какая модель минимизирует AIC/BIC в большинстве случаев.
2. Оценить распределение лучших моделей по субъектам.
3. Проверить наличие кластеризации по типу распределения.

---

# Связь с предыдущими этапами

Для параметров лучшей модели проверить корреляции с:

* Speed Axis
* Lateral Axis
* Residual Axis
* Microdynamic parameters
* Tail Geometry metrics

Цель:

Понять, является ли тип распределения:

* функцией trait;
* функцией state;
* самостоятельной характеристикой.

---

# Возможные интерпретационные сценарии

1. Lognormal доминирует → мультипликативная модель шума.
2. Ex-Gaussian доминирует → базовый быстрый процесс + редкие задержки.
3. Gamma доминирует → накопительный механизм до порога.
4. Смесь моделей → наличие подтипов или state-переключений.

Важно:

Выбор модели не должен базироваться исключительно на AIC.
Необходимо учитывать устойчивость параметров и межсубъектную воспроизводимость.

---

# Критерии завершения Task 35.1

1. Проведено сравнение моделей для всех последовательностей.
2. Определена популяционная доминантная модель (если существует).
3. Проверена устойчивость параметров.
4. Выполнен анализ связи с trait/state слоями.

---

# Архитектурное значение

Результаты Task 35.1 позволят:

* уточнить генеративную модель RT;
* отличить heavy-tail как масштабный эффект от state-провалов;
* определить, следует ли в v5 использовать параметрические индексы распределения;
* подготовить основу для следующего этапа Framework (Continuum vs Discrete Types).

---

Task 35.1 завершает углублённый анализ распределений в рамках Stage 6 без нарушения структуры Exploratory Architecture Framework v4.
