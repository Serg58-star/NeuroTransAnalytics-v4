# Task 34 — Stage 5: Microdynamic Architecture (Внутритестовая динамика)

## Контекст

Task 34 реализует **ЭТАП 5 из Exploratory Architecture Framework v4**.

После завершения структурной верификации (Stages 1–5 геометрического блока) установлено:

* Латентное пространство 7 признаков устойчиво, линейно и трёхмерно.
* Оси: Уровень Скорости, Латерализация, Неспецифический Тонус.
* Геометрия инвариантна к масштабу, канальной агрегации и конструктивным параметрам.

Однако все предыдущие этапы анализировали агрегированное пространство.

Stage 5 Framework проверяет:

> существует ли самостоятельная **динамическая архитектура внутри одного теста (36 стимулов)**,
> способная формировать дополнительную размерность или state-компонент.

---

# Методологические ограничения

1. Используются trial-level данные (36 попыток) без изменения генеративной логики.
2. Production-признаки не пересчитываются.
3. Анализ проводится параллельно агрегированному пространству.
4. Никакие новые производные индексы не заменяют базовые 7 признаков.
5. Все выводы фиксируются операционально.

Stage 5 — это анализ временной структуры, а не изменение геометрии признаков.

---

# Блок XX — Блочная декомпозиция (36 → 3×12)

Разделить каждую сессию на три блока:

* Block 1: 1–12
* Block 2: 13–24
* Block 3: 25–36

Для каждого блока вычислить:

* медиану RT
* MAD
* ΔV4, ΔV5 (если применимо)

Проверить:

1. Наличие тренда ускорения/замедления.
2. Изменение латерализации по блокам.
3. Изменение третьей компоненты (Residual Axis).

Методы:

* линейная регрессия RT ~ block_index
* repeated measures ANOVA (если допустимо)
* эффект размера (Cohen’s d)

Цель:

Выявить истощение, мобилизацию или адаптацию внутри одной сессии.

---

# Блок XXI — Autocorrelation RT

Для каждой сессии рассчитать:

* ACF (lag 1–5)
* Ljung–Box test

Проверить:

1. Существование краткосрочной памяти в RT.
2. Наличие устойчивых slow/fast периодов.

Цель:

Определить, является ли поведение последовательности стационарным
или содержит внутреннюю временную зависимость.

---

# Блок XXII — Burst-анализ

Определить burst как последовательность из ≥3 попыток,
где RT стабильно ниже индивидуальной медианы − 1 MAD
(или выше медианы + 1 MAD — slow burst).

Вычислить:

* частоту burst
* среднюю длину burst
* асимметрию fast vs slow burst

Проверить:

1. Формирует ли burst-поведение дополнительную ось вариативности.
2. Связан ли burst с латерализацией.

Цель:

Выявить фазовую организацию производительности.

---

# Блок XXIII — Пермутационный тест порядка стимулов

Для каждой сессии:

1. Перемешать порядок 36 RT (1000+ пермутаций).
2. Пересчитать:

   * автокорреляцию
   * блоковые различия
   * burst-частоту

Сравнить с оригинальной последовательностью.

Цель:

Проверить, играет ли фиксированная последовательность стимулов
структурную роль в формировании динамики.

---

Дополнительно:

* доля субъектов с автокоррелированной структурой
* распределение burst-параметров
* связь микро-параметров с 3D-координатами

---

# Критерии анализа

1. Формирует ли микродинамика дополнительную размерность?
2. Является ли она общей для популяции или индивидуальной?
3. Встраивается ли она в существующие 3 оси?
4. Разрушается ли динамика при пермутации порядка?

---

# Возможные сценарии

1. Динамика отсутствует → тест стационарен.
2. Динамика есть, но не формирует новую ось.
3. Burst и autocorrelation создают state-компонент.
4. Порядок стимулов критичен для структуры.

---

# Формат отчёта

1. Сводная таблица по всем блокам.
2. Графики динамики внутри теста.
3. Связь микро-показателей с тремя базовыми осями.
4. Аналитический раздел:

   * существует ли динамическая архитектура;
   * влияет ли она на проектирование v5;
   * следует ли менять длину теста или порядок стимулов.

---

Stage 5 считается завершённым,
если внутренняя динамика полностью проанализирована
и сопоставлена с латентной 3D-геометрией.
