# Task 31 — Stage 3: Канальная реконструкция (Parvo / Magno / Koniocellular)

## Контекст

Stage 3 реализует следующий этап Exploratory Architecture Framework v4.

После Stage 1 и Stage 2 установлено:

* латентная геометрия аддитивна и порядково-инвариантна;
* абсолютная скорость — самостоятельная структурная ось;
* латерализация образует ортогональную ось.

Stage 3 проверяет:

> воспроизводится ли латентная структура при переходе
> от тестового базиса (simple / color / shift)
> к функционально-канальному представлению.

---

# Методологические ограничения

1. Используется тот же production-набор из 7 признаков.
2. Trial-level данные не модифицируются.
3. BaselineFeatureExtractor не изменяется.
4. stability.py используется без изменений.
5. Between/Within рассчитывается строго по формуле Task 27.3J.
6. Канальные индексы строятся как линейные комбинации уже существующих признаков.

Stage 3 — это смена представления, а не генеративная реконструкция данных.

---

# Концепция канальной реконструкции

В рамках текущей тестовой архитектуры предполагается следующая операциональная декомпозиция:

* **Parvo-index** — компоненты, преимущественно связанные с цветовой обработкой (V4 residual-компоненты);
* **Magno-index** — компоненты, связанные с динамикой и движением (V5 residual-компоненты);
* **Speed-axis** — общая скорость (Mean-компоненты);
* **Koniocellular proxy** — при наличии цвето-специфических признаков (если применимо).

Важно: каналы интерпретируются как операциональные индексы, а не как физиологическое доказательство.

---

# Блок A — Построение канальных индексов

Сформировать:

* Parvo_mean = среднее(V4_left_residual, V4_right_residual)
* Magno_mean = среднее(V5_left_residual, V5_right_residual)
* Parvo_diff = разность(V4_left_residual, V4_right_residual)
* Magno_diff = разность(V5_left_residual, V5_right_residual)

Сформировать пространство:

[Parvo_mean, Magno_mean, Parvo_diff, Magno_diff, asym_dv1_abs, asym_dv1_rel, psi_tau]

Выполнить:

* PCA
* Bootstrap
* Split-half
* Between/Within

Сравнить с baseline и Stage 2.

---

# Блок B — Сжатое канальное пространство

Сформировать агрегированное пространство:

* Parvo_index (Mean V4)
* Magno_index (Mean V5)
* Lateral_index (комбинация Diff + asym)

Провести PCA и stability-анализ.

Цель:

Определить, сохраняется ли двумерная/трёхмерная структура при явной канальной агрегации.

---

# Блок C — Корреляционная матрица каналов

Построить:

Corr(Parvo_mean, Magno_mean)
Corr(Parvo_mean, Lateral_index)
Corr(Magno_mean, Lateral_index)

Оценить:

* ортогональность каналов;
* наличие доминирующего канала;
* перераспределение дисперсии.

# Критерии анализа

1. Сохраняется ли размерность ≥ 2?
2. Формирует ли Parvo/Magno независимые оси?
3. Изменяется ли вклад латерализации?
4. Стабильна ли структура при канальной агрегации?

---

# Возможные сценарии

1. Parvo и Magno образуют две устойчивые оси.
2. Один канал доминирует, второй вторичен.
3. Канальная агрегация разрушает структуру (редукция размерности).

Каждый сценарий фиксируется без физиологической гипер-интерпретации.

---

# Формат отчёта

1. Сравнительная таблица метрик.
2. Таблица loadings.
3. Корреляционная матрица каналов.
4. Аналитический раздел:

   * структура каналов;
   * устойчивость;
   * влияние на проектирование будущей системы.
