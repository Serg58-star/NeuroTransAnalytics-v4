# Task 32 — Stage 4: Стимул-конструктивная декомпозиция (PSI / Shift / Position / Mask)

## Контекст

Stage 4 продолжает Exploratory Architecture Framework v4.

После Stage 1–3 установлено:

* Геометрия аддитивна и порядково-инвариантна.
* Абсолютная скорость — структурная ось.
* Латерализация — ортогональная независимая ось.
* Простая канальная агрегация (Parvo/Magno) разрушает геометрию.

Stage 4 проверяет принципиальный вопрос:

> Является ли третья (и потенциально более высокая) размерность
> следствием конструктивных параметров стимула?

Иными словами — скрыта ли часть геометрии в:

* PSI (предстимульный интервал),
* shift_parameter,
* позиции стимула в серии,
* структуре маски.

---

# Методологические ограничения

1. Используется тот же production-набор признаков (7 признаков).
2. Trial-level данные не модифицируются.
3. BaselineFeatureExtractor не изменяется.
4. stability.py используется без изменений.
5. Between/Within рассчитывается строго по формуле Task 27.3J.
6. Все стратификации выполняются через фильтрацию или группировку,
   без изменения способа расчёта признаков.

Stage 4 — это декомпозиция пространства по условиям,
а не перестройка признаков.

---

# Структура Stage 4

## Блок A — PSI-стратификация

Разбить данные по уровням PSI.

Для каждого уровня PSI:

* построить PCA
* рассчитать Dim
* PC1 %
* PR
* Bootstrap SD
* Split-half
* Between/Within

Сравнить размерность между уровнями PSI.

Цель:

Проверить, меняется ли латентная размерность при изменении предстимульного интервала.

---

## Блок B — Нелинейность PSI

Построить зависимость:

* ΔV4 ~ PSI
* ΔV5 ~ PSI

Проверить:

* линейную модель
* экспоненциальную модель
* сигмоидальную модель
* change-point (если возможно)

Цель:

Выявить, присутствует ли пороговая или нелинейная динамика.

---

## Блок C — Shift_parameter (0–3)

Стратифицировать данные по shift_parameter.

Для каждого значения shift:

* построить PCA
* оценить Dim
* сравнить вклад V5-компонент

Цель:

Определить, усиливает ли динамический стимул (shift>0)
третью размерность пространства.

---

## Блок D — Позиционная динамика

Разделить тест на сегменты (например, 1–12, 13–24, 25–36).

Для каждого сегмента:

* построить PCA
* сравнить Dim и PC1 %

Цель:

Проверить, меняется ли геометрия по мере прохождения теста.

---

## Блок E — Mask complexity (если операционализируемо)

Если возможно количественно оценить сложность mask_triples:

* ввести индекс сложности (энтропия / число переходов / симметрия)
* стратифицировать данные
* оценить изменение размерности

---

# Метрики к отчёту

| Condition | Dim | PC1 % | PR | Boot SD | SH ΔPC1 | B/W |
| --------- | --- | ----- | -- | ------- | ------- | --- |

Дополнительно:

* сравнение loadings по условиям
* изменение доли V4 / V5 вкладов
* изменение вклада латерализации

---

# Критерии анализа

1. Появляется ли дополнительная размерность при стратификации?
2. Усиливается ли вклад V5 при shift>0?
3. Снижается ли межсубъектная ось при коротком PSI?
4. Происходит ли изменение структуры по ходу теста?

---

# Возможные сценарии

1. Размерность инвариантна → конструктивные параметры вторичны.
2. Размерность растёт при определённых PSI или shift → выявляется скрытая ось.
3. Геометрия зависит от сегмента теста → присутствует динамическая компонента.

Все выводы фиксируются строго операционально,
без физиологической гиперинтерпретации.

---

# Формат отчёта

1. Сводная таблица метрик по всем условиям.
2. Таблицы loadings по ключевым условиям.
3. Аналитический раздел:

   * влияние PSI;
   * влияние shift;
   * влияние позиции;
   * влияние сложности маски;
   * значение для проектирования будущей системы.
