# Task_27_2C_Финальная_Стабилизация_Регрессионного_Инструмента

## Цель

Перед запуском production-run устранить оставшиеся методические недочёты
в модуле symmetric_regression.py и окончательно стабилизировать
регрессионный инструмент.

Работа выполняется на реальных данных.
Без PCA.
Без кластеризации.
Без визуализации.
Только числовая проверка корректности модели.

---

# I. Стандартизация — уточнение логики

1. Зафиксировать в коде и отчёте:

   - Выполняется ли стандартизация отдельно по каждому полю
     (left/right/center),
     или на объединённом массиве.

2. В отчёте явно указать:

   - тип стандартизации,
   - размер выборки,
   - среднее и стандартное отклонение по каждому предиктору.

3. Обеспечить сопоставимость standardized β между left и right.

---

# II. Гетероскедастичность — обязательная робастная оценка

1. Если тест Бреуша–Пагана даёт p < 0.05:

   - Пересчитать модель с робастными стандартными ошибками (HC3).
   - Вывести обе версии коэффициентов:
        • классические SE
        • робастные SE

2. В отчёте указать:

   - влияет ли гетероскедастичность на значимость β.

3. Не ограничиваться рекомендацией "consider robust" —
   реализовать расчёт фактически.

---

# III. Дополнительная проверка устойчивости PSI_tau

Добавить:

1. Проверку зависимости τ от диапазона PSI:

   - Рассчитать range(PSI) для каждого субъекта.
   - Рассчитать корреляцию:
         τ ~ range(PSI)

2. Если |r| > 0.3 —
   указать, что τ частично определяется диапазоном PSI.

3. Добавить минимальный критерий диапазона PSI:

   - Исключать субъекты с range(PSI) < заранее заданного порога.

4. В отчёте указать:
   - сколько субъектов исключено,
   - влияет ли это на распределение τ.

---

# IV. Практическая значимость ΔR²

В дополнение к F-test:

1. Явно вывести:

   - ΔR²
   - p-value F-test

2. Добавить интерпретацию:

   - Если ΔR² < 0.02 и p < 0.05 →
     статистически значимо, но практически незначимо.

3. Отдельно отметить случаи:

   - ΔR² ≥ 0.05 → существенный вклад MAD.

---

# V. Проверка распределений признаков

Перед регрессией добавить блок:

1. Для каждого признака рассчитать:

   - Skewness
   - Kurtosis
   - Количество выбросов (>3 SD)

2. Если выбросы > 5% наблюдений:

   - указать риск искажения регрессии.

3. В отчёте явно вывести:

   - распределение ΔV4
   - распределение ΔV5
   - распределение τ

---

# VI. Финальный отчёт

Подготовить документ:

    Task_27_2C_Отчёт_Финальной_Стабилизации.md

Отчёт должен содержать:

1. Подтверждение корректности стандартизации.
2. Результаты тестов гетероскедастичности.
3. Робастные коэффициенты (если применимо).
4. Устойчивость τ (включая range PSI).
5. ΔR² с интерпретацией практической значимости.
6. Анализ распределений и выбросов.
7. Итоговое заключение:

   - Инструмент методически стабилен.
   - Или требуется дополнительная коррекция.

---

# Ограничения

- Не запускать production-run.
- Не выполнять PCA.
- Не выполнять кластеризацию.
- Не строить графики.

Цель — довести инструмент до полностью стабильного состояния
перед запуском на полном массиве.

Конец документа.
