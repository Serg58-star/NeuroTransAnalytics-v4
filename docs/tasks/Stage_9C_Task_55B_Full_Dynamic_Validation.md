# Stage 9C — Task 55B
# Полная динамическая валидация кандидатов κ

## Статус
АРХИТЕКТУРНАЯ КОРРЕКЦИЯ (Stage 2 / 3)

## Основание

Stage 9C Task 55A (T = 10) выявил 3 кандидата κ:

κ ∈ {0.08, 0.09, 0.10}

Необходимо выполнить полную динамическую валидацию
и выбрать минимальное κ, удовлетворяющее всем критериям устойчивости.

---

# 1. Ограничения (НЕ НАРУШАТЬ)

Запрещено:

- изменять Z-space
- изменять Severity формулу
- изменять Anchored Projection
- изменять MAD-стандартизацию
- пересчитывать MCD
- выполнять bootstrap

Изменяется только продольный дрейф (κ уже встроен).

---

# 2. Вычислительные параметры

- N = 400
- T = 10
- Без bootstrap
- Eigendecomposition разрешена
- Silhouette рассчитывать один раз на κ

---

# 3. Проверяемые критерии

Для каждого κ выполнить:

## 3.1 Переходная матрица

- Построить 4×4 Transition Matrix
- Проверить:

  - P_ii ≤ 0.95 (нет absorbing states)
  - строки суммируются к 1

---

## 3.2 Спектральный анализ

- Вычислить собственные значения
- Проверить:

  - |λ_max| = 1 (допуск 1e-6)
  - Нет |λ| > 1
  - Spectral_Gap ≥ 0.1

---

## 3.3 Стационарное распределение

- Stationary_Q2 ≤ 50%

---

## 3.4 Энтропия переходов

- Mean_System_Entropy ≥ 0.5

---

## 3.5 Динамическая устойчивость Severity

Проверить:

- Saturation_Slope ∈ [-0.05, 0.02]
- Нет экспоненциального ускорения ΔS

---

## 3.6 Геометрическая непрерывность (предварительная)

- Silhouette_longitudinal < 0.20

(Это предварительная проверка, финальная геометрическая валидация будет в Task 55C.)

---

# 4. Логика выбора κ

Выбрать минимальное κ,
удовлетворяющее всем критериям.

Принцип:

минимально достаточная стабилизация.

---

# 5. Failure Criteria

Если ни один κ не проходит:

→ вернуть отчёт
→ инициировать пересмотр функции g(S)
