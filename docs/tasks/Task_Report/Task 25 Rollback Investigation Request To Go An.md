# Task 25 — Architectural Violation Investigation & Rollback Request

## 1. Контекст

В рамках реализации Task 25 (Scenario A0.3 — Architectural Symmetry) было зафиксировано изменение слоя C3.3 (QC & Aggregation).

Согласно утверждённой спецификации Task 25:

> Изменения C3.1–C3.3 запрещены.

Факт изменения C3.3 нарушает архитектурную изоляцию C3-пайплайна и статус стабилизации, зафиксированный в C3 Phase Completion Summary.

Настоящий документ инициирует расследование и потенциальный rollback.

---

## 2. Требуемые подтверждения

### 2.1 Статус поля stimulus_location

Подтвердить:

1. Присутствовало ли поле `stimulus_location` в AggregatedFrame ДО внесённых изменений?
2. Если да — почему потребовалось изменение C3.3?
3. Если нет — на каком уровне пайплайна оно терялось?

---

### 2.2 Объём изменений в C3.3

Предоставить:

1. Конкретный diff изменений в `qc_aggregation_v4.py`.
2. Перечень изменённых group_cols.
3. Подтверждение, повлияли ли изменения на:

   * A0.0
   * A0.1
   * A0.2

---

### 2.3 Возможность отката

Подтвердить:

1. Можно ли полностью откатить изменения C3.3 без потери функциональности A0.3?
2. Возможно ли реализовать A0.3 исключительно в C3.4, используя существующую структуру AggregatedFrame?

---

## 3. Архитектурное требование

Если изменение C3.3 не является строго необходимым:

* выполнить rollback изменений C3.3;
* восстановить первоначальный контракт C3.3;
* реализовать A0.3 исключительно на уровне C3.4.

Если изменение действительно необходимо:

* сформировать отдельную задачу на контролируемое расширение C3.3;
* описать новый контракт AggregatedFrame;
* пройти повторную архитектурную валидацию.

---

## 4. Ожидаемый формат ответа

Ответ должен содержать:

1. Подтверждение по пункту 2.1.
2. Конкретный diff или описание изменений.
3. Оценку влияния на существующие сценарии.
4. Предложение: rollback или controlled extension.

До получения ответа Task 25 считается:

> Архитектурно неподтверждённой.
