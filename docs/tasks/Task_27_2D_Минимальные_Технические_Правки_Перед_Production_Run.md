# Task_27_2D_Минимальные_Технические_Правки_Перед_Production_Run

Корректировки касаются:

1. Численной устойчивости робастной оценки (HC3).
2. Гарантированной синхронизации индексов при анализе зависимости τ.

# I. Замена np.linalg.inv → np.linalg.pinv (HC3)

## Проблема

В блоке расчёта робастной ковариационной матрицы используется:

    XtX_inv = np.linalg.inv(X_with_intercept.T @ X_with_intercept)

При высокой мультиколлинеарности или почти вырожденной матрице
возможен LinAlgError или численная нестабильность.

## Требуется

1. Заменить:

    np.linalg.inv(...)

   на:

    np.linalg.pinv(...)

2. Зафиксировать в комментарии:

   "Using Moore–Penrose pseudo-inverse for numerical stability"

3. Убедиться, что расчёт β и SE не меняет логическую структуру отчёта.

# II. Синхронизация индексов при анализе τ

## Проблема

В анализе устойчивости τ используются:

- tau_values (по features_df)
- subject_psi_counts (через цикл по index features_df)
- psi_ranges

Необходимо гарантировать, что:

- порядок субъектов строго совпадает
- индексы не теряются
- корреляция рассчитывается на согласованных массивах

## Требуется

1. Явно сформировать DataFrame:

    tau_analysis_df = pd.DataFrame({
        "tau": ...,
        "psi_count": ...,
        "psi_range": ...
    }, index=features_df.index)

2. Применять фильтрацию через:

    tau_analysis_df.loc[valid_indices]

   а не через независимые массивы.

3. Перед расчётом корреляции:

   проверить совпадение длин и индексов.

4. В отчёте указать:

   - количество валидных субъектов
   - количество исключённых субъектов
