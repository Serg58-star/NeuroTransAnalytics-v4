# Task_27_3B_Minimal_Diagnostic_Feature_Extraction

## Цель

Выявить точную причину,
по которой BaselineFeatureExtractor возвращает 0 субъектов
при загрузке реальных данных из neuro_data.db.

Запуск выполняется только в диагностическом режиме
на минимальном наборе данных (1 субъект).

Никаких массовых прогонов.
Никаких silent-fail.
Никаких подавлений исключений.

---

# I. Ограничения

Запрещено:

- Глушить исключения через try/except continue
- Выполнять анализ на полном массиве
- Делать предположения без фактической ошибки
- Переходить к Task 27.3 до завершения диагностики

Разрешено:

- Выводить traceback
- Печатать структуру DataFrame
- Логировать промежуточные данные

# II. Диагностический сценарий

Из trials_df выбрать:

    test_subject = trials_df['subject_id'].unique()[0]

- количество trial
- список колонок

    • color (если есть)
