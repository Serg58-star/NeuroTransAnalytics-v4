# Task_27_2B_Методическая_Доработка_Перед_Production_Run

## Цель

Перед запуском production-run устранить выявленные методические недоработки
в симметричном регрессионном анализе.

Работа выполняется на реальных данных.
Без визуализации.
Без PCA.
Без кластеризации.
Только числовая проверка корректности модели.

---

# I. Стандартизация переменных

## 1.1. Проблема

В текущей версии регрессии признаки могут находиться в разных масштабах,
что затрудняет интерпретацию β-коэффициентов.

## 1.2. Требуется

1. Выполнить стандартизацию (z-score) всех предикторов:
   - Median_ΔV1_field
   - MAD_ΔV1_field

2. Построить модели на стандартизированных переменных.

3. В отчёте вывести:
   - Standardized β
   - R²
   - Adjusted R²

4. Указать различия между нестандартизированной и стандартизированной моделью.

---

# II. Проверка гетероскедастичности

## 2.1. Требуется

Для каждой линейной модели:

1. Выполнить тест Бреуша–Пагана
   или эквивалентную проверку гетероскедастичности.

2. Рассчитать:

   - p-value теста
   - зависимость |residual| от предиктора

3. Если p < 0.05 —
   указать наличие гетероскедастичности.

4. В случае выявления —
   дополнительно оценить модель с робастными стандартными ошибками.

---

# III. Проверка устойчивости PSI_tau

## 3.1. Требуется

1. Для каждого субъекта определить:
   - количество PSI-наблюдений,
   - диапазон PSI.

2. Исключить субъекты,
   у которых:
   - число PSI-точек < 4
   или
   - экспоненциальная аппроксимация неустойчива.

3. Рассчитать:

   - распределение τ
   - коэффициент вариации τ

4. Проверить корреляцию τ с:
   - количеством PSI-наблюдений
   - диапазоном PSI

Если τ зависит от числа точек —
указать это как ограничение.

---

# IV. ΔR² и вклад MAD

## 4.1. Требуется

Для каждой множественной модели:

1. Рассчитать:

   - R² базовой модели (только Median_ΔV1)
   - R² полной модели (Median_ΔV1 + MAD)

2. Рассчитать ΔR².

3. Выполнить F-test для сравнения вложенных моделей.

4. Если ΔR² < 0.02 —
   указать, что вклад MAD минимален.

5. Если ΔR² значим —
   зафиксировать вклад вариабельности как независимую компоненту.

---

# V. Итоговый отчёт

Подготовить документ:

    Task_27_2B_Отчёт_О_Методической_Доработке.md

В отчёте должны быть:

1. Таблицы стандартизированных коэффициентов.
2. Результаты теста гетероскедастичности.
3. Устойчивость τ.
4. ΔR² для каждой модели.
5. Итоговое заключение по каждому признаку:

   - Надёжен.
   - Частично зависим.
   - Методически нестабилен.
   - Требует исключения.

---

# Ограничения

- Не запускать PCA.
- Не запускать кластеризацию.
- Не проводить визуализацию.
- Не переходить к production-run.

Цель — полностью стабилизировать модель
перед запуском на полном массиве.

Конец документа.
