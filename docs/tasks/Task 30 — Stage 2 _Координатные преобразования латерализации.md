# Task 30 — Stage 2: Координатные преобразования латерализации

## Контекст

Task 30 реализует **ЭТАП 2 (Координатные преобразования)**
из документа *Exploratory Architecture Framework v4*.

Stage 1 установил:

* геометрия аддитивна;
* структура порядково-инвариантна;
* межсубъектная абсолютная позиция является структурной осью.

Stage 2 проверяет:

> формирует ли латерализация самостоятельную латентную ось
> и сохраняется ли геометрия при переходе к альтернативным координатным представлениям.

---

# Методологические ограничения

1. Используется тот же production-набор из 7 признаков.
2. Trial-level данные не изменяются.
3. BaselineFeatureExtractor не модифицируется.
4. stability.py используется без изменений.
5. Between/Within рассчитывается тем же методом, что в Task 27.

Stage 2 — это координатное преобразование уже существующего пространства,
а не изменение генеративной модели.

---

# Цели Stage 2

1. Отделить общую скорость от асимметрии.
2. Проверить существование независимой оси латерализации.
3. Оценить вклад латерализации в PC1 и PC2.
4. Проверить устойчивость размерности при смене координат.

---

# Блок A — Преобразование (L + R)/2 и (L − R)

Для residual-компонент V4 и V5 построить:

* Mean_V4 = (V4_left + V4_right) / 2
* Diff_V4 = (V4_left − V4_right)
* Mean_V5 = (V5_left + V5_right) / 2
* Diff_V5 = (V5_left − V5_right)

Asym_dv1 и psi_tau оставить без изменений.

Сформировать новое пространство признаков.

Выполнить:

* PCA
* Bootstrap
* Split-half
* Between/Within

Сравнить с baseline.

---

# Блок B — PCA только по асимметриям

Сформировать подпространство:

* Diff_V4
* Diff_V5
* asym_dv1_abs
* asym_dv1_rel

Выполнить:

* PCA
* Bootstrap
* Split-half
* Between/Within

Цель:
Проверить, формирует ли асимметрия самостоятельную устойчивую геометрию.

---

# Блок C — Центр vs Периферия (если применимо)

Если структура признаков позволяет:

* выделить компоненты, отражающие центр,
* сравнить их с периферическими.

Выполнить PCA для каждого подпространства отдельно.

# Метрики, обязательные к отчёту

| Transformation | Dim | PC1 % | PR | Boot SD | SH ΔPC1 | B/W |
| -------------- | --- | ----- | -- | ------- | ------- | --- |

Дополнительно:

* Loadings PC1 и PC2 для baseline и Mean/Diff-преобразования.
* Доля дисперсии, объясняемая Diff-компонентами.

---

# Критерии анализа

1. Сохраняется ли Dim ≥ 2?
2. Появляется ли отдельная «асимметрическая» компонента?
3. Уменьшается ли вклад межсубъектной абсолютной позиции?
4. Стабильна ли геометрия при смене координат?

---

# Ожидаемые сценарии

1. Латерализация формирует вторую ось.
2. Латерализация вторична по отношению к общей скорости.
3. Асимметрия не образует устойчивой геометрии.

Каждый сценарий фиксируется без интерпретационных гипотез.

---

# Формат отчёта

1. Сравнительная таблица метрик.
2. Таблица нагрузок (loadings).
3. Краткий аналитический раздел:

   * роль латерализации;
   * изменение структуры осей;
   * влияние координатной смены на размерность.

---

Stage 2 считается завершённым,
если координатное преобразование полностью проанализировано
и сравнено с baseline без архитектурных нарушений.
