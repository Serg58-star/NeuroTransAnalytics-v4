ОБЯЗАТЕЛЬНО К ИСПОЛНЕНИЮ
Разрешено после завершения Stage 9D.1

---

# 1. Цель этапа

Выполнить полную структурную проверку эмпирических данных в Z-пространстве в соответствии с протоколом Stage 9D.

Этап направлен на проверку:

* устойчивости Z-space на реальных данных
* отсутствия выраженной демографической сегрегации
* отсутствия геометрического дрейфа по полу и возрасту
* сохранения инвариантов Core v5

---

# 2. Архитектурные ограничения (строго)

Core v5 — LOCKED.

Запрещено:

* менять κ (κ = 0.08 фиксировано)
* менять формулу Severity
* пересчитывать Anchored thresholds
* вводить демографическую калибровку
* изменять матрицу Σ вне процедуры робастной оценки

Формула Severity:

S(Z) = Zᵀ Σ⁻¹ Z

Центроид не вычитается.
Относительный дрейф центроида не используется.

---

# 3. Предварительный структурный блок (обязательный)

Перед запуском демографических тестов необходимо:

## 3.1 Определить N субъектов

* Подсчитать количество уникальных subject_id.
* Зафиксировать:

  * общее число субъектов (N)
  * среднее число тестов на одного субъекта
  * медиану числа тестов
  * минимум и максимум

## 3.2 Проверить распределение числа тестов на одного субъекта

* Построить распределение количества trials на subject_id.
* Зафиксировать долю субъектов с 1 тестом.
* Зафиксировать долю субъектов с ≥3 тестами.
* Проверить наличие выраженной асимметрии распределения.

Интерпретация запрещена — только фиксация структуры.

---

# 4. Расчёт Z-пространства (эмпирический слой)

## 4.1 Робастная стандартизация

Для каждого канального показателя:

* вычислить медиану
* вычислить MAD
* выполнить стандартизацию

Z = (X − Median) / MAD

Использовать исключительно робастную оценку масштаба.

## 4.2 Проверка геометрии

Вычислить:

* Participation Ratio (PR)
* Effective Rank
* PC1 explained variance
* Silhouette (k = 2..5)

Критерии соответствия — строго по Task 50A.

---

# 5. Вычисление Severity

Использовать робастную ковариационную матрицу Σ.

Вычислить:

S(Z) = Zᵀ Σ⁻¹ Z

Проверить:

* положительную полуопределённость Σ
* устойчивость condition number

Центроид не используется.

---

# 6. Sex-Differentiation Block

## 6.1 Различие центров

Δ_sex = || μ_group1 − μ_group2 ||₂

Сравнить с критериями PASS/FAIL Stage 9D.

## 6.2 Различие ковариаций

Оценить относительное расхождение ковариационных структур.

## 6.3 Различие распределения Severity

Рассчитать Cohen’s d.

## 6.4 Silhouette при разметке по полу

Проверить наличие кластеризации.

---

# 7. Age-Gradient Block

## 7.1 Корреляция Severity ~ Age

Оценить ρ (robust).

## 7.2 Регрессия (кластер-робаст)

Модель:

Severity ~ Age

с корректировкой на кластеризацию по subject_id.

## 7.3 Нелинейность

Проверить вклад нелинейной модели (например, spline).

## 7.4 Геометрический дрейф по квартилям возраста

Разделить выборку на возрастные квартили.

Проверить изменения:

* PR
* PC1
* центров
* ковариаций

---

# 8. Interaction Block (Sex × Age)

Построить модель:

Severity ~ Sex + Age + Sex×Age

Использовать:

* cluster-robust SE
  или
* mixed-effects (random intercept по subject_id)

Оценить значимость interaction-термина.

---

# 9. Кластер-робаст модель (обязательное требование)

Все регрессионные оценки обязаны учитывать:

* повторные тесты одного субъекта
* кластеризацию по subject_id

Игнорирование кластеризации запрещено.
