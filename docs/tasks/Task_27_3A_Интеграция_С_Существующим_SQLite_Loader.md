# Task_27_3A_Интеграция_С_Существующим_SQLite_Loader

## Цель

Реализовать production-run (Task 27.3) через прямую интеграцию
с существующей инфраструктурой загрузки SQLite-базы
в проекте NeuroTransAnalytics-v4.

База данных:

    C:\NeuroTransAnalytics-v4\neuro_data.db

---

# I. Принципиальные требования

1. Не реализовывать новый SQLite loader.
2. Не дублировать схему таблиц вручную.
3. Не угадывать структуру базы.
4. Использовать существующие модули загрузки данных
   из проекта NeuroTransAnalytics-v4.

---

# II. Действия

1. Проанализировать код основного проекта
   в каталоге:

       C:\NeuroTransAnalytics-v4

2. Найти:
   - модуль загрузки SQLite
   - описание схемы таблиц
   - функцию/класс, формирующий trial-level DataFrame

3. Использовать этот механизм в exploratory_lab,
   не копируя код,
   а импортируя его.

4. Убедиться, что загрузка выполняется:

   - read-only
   - без модификации базы
   - без создания временных таблиц

---

# III. Требования к загрузке данных

Loader должен возвращать DataFrame,
содержащий как минимум:

- subject_id
- field (left/right/center)
- RT
- PSI
- test_type
- признаки, необходимые для расчёта:
    - ΔV1
    - ΔV4
    - ΔV5
    - MAD

Никакой предварительной агрегации.

---

# IV. Верификация

Перед запуском Task_27_3 необходимо:

1. Вывести:
   - количество субъектов
   - количество trial
   - количество записей по каждому полю
2. Убедиться в отсутствии пропусков критических колонок.
3. Подготовить краткий отчёт:

    Task_27_3A_Отчёт_Об_Интеграции_SQLite.md

---

# V. Ограничения

- Не выполнять production-run до подтверждения корректной интеграции.
- Не создавать альтернативную схему загрузки.
- Не использовать synthetic data.
- Не экспортировать данные в CSV.

---

Цель — использовать реальную инфраструктуру проекта
и обеспечить воспроизводимый production-run.
