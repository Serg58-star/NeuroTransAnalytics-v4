# Chat_Summary_Phase_Exploratory_Bootstrap

**Project:** NeuroTransAnalytics-v4  
**Phase:** Exploratory Bootstrap (C3.x + GUI Integration)  
**Status:** Stable, Interactive, Synthetic-first validated  
**Date:** 2026-02  

---

## 1. Контекст фазы

Данный этап проекта является инженерным bootstrap-этапом, а не реализацией финального исследовательского анализа.

В рамках этой фазы были:

- сформированы архитектурные уровни C2–C4,
- реализован слой C3.x (Exploratory Procedures),
- создана система артефактов,
- построен рабочий PySide6 GUI,
- обеспечена интерактивность запуска процедур из интерфейса,
- подтверждена воспроизводимость и разделение слоёв.

Фаза выполнена на synthetic-first подходе, без подключения реальных данных.

---

## 2. Архитектурная модель (текущее состояние)

### C2 — Data Architecture

- SQLite (`neuro_data.db`) — canonical store.
- Документация синхронизирована с отказом от Excel.
- Физическая реализация не смешивается с логической моделью.

---

### C3 — Deterministic Computation

- ETL, QC, Scenario computation.
- Никакой интерпретации.
- Результаты сохраняются в виде артефактов.

---

### C3.x — Exploratory Procedures

Реализованы:

1. `MultimodalityDetection v1.1.0`
2. `ChangePointDetection v1.1.0`

Принципы:

- фиксированная версия процедуры,
- явные гиперпараметры,
- запись provenance,
- обязательная non-interpretation clause,
- воспроизводимость через seed.

---

### C3.5 — Visualization

- Read-only.
- Нет вычислений.
- Отображает precomputed artifacts.
- Matplotlib интегрирован в PySide6.

---

### Persistence Layer

- JSON-артефакты в `artifacts/exploratory/`
- Детеминированный SHA-хеш параметров
- Полная реконструкция результата из файла

---

## 3. GUI (Task 15 завершён)

GUI перешёл из режима просмотра в режим interactive execution layer.

Реализовано:

- Parameter panels для процедур
- `Run Procedure` → вызов C3.x через controller
- Автоматическое сохранение JSON
- Авто-обновление списка артефактов
- Отображение графика + provenance
- Строгая архитектурная изоляция GUI от C3.x

GUI функционально стабилен.

## 6. Ключевая цель проекта

Главная задача:

> Используя существующий массив данных NeuroTransAnalytics,  
> выявить закономерности архитектуры зрительных реакций,  
> которые можно использовать при проектировании нового поколения тестирующей программы.

Любое дальнейшее развитие должно:

- работать с реальными данными,
- выявлять структурные закономерности,
- иметь методологическое значение,
- приближать к проектированию следующей версии системы.
