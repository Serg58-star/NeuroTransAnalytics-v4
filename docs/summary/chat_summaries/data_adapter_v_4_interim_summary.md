## 3. Архитектурные инварианты v4

1. `cnt` — единственный идентификатор тестовой сессии во всех слоях данных.
2. Модели `Session`, `StimulusEvent`, `ResponseEvent` согласованы по `session_id`.
3. Warmup-логика:
   - ровно 3 независимых предтестовых блока (`order = 1, 2, 3`);
   - PSI не накапливается между блоками;
   - warmup-контекст (цвет, позиция, test_triple) фиксируется явно.
4. `ResponseEvent.validity_flag`:
   - поле присутствует в модели;
   - значения ещё не вычислялись и равны `None`.

### Уточнение порога валидации RT

Минимально допустимое время реакции зафиксировано на уровне **135 мс**.

Данное значение соответствует параметру `MinRedLight`, изменённому
в программе тестирования **25.12.2016** и отражённому в `config_old.ini`:

MinRedLight = 135


Тестирующая программа не регистрировала реакции быстрее указанного порога,
поэтому использование значения **135 мс** в аналитическом валидаторе
обеспечивает **полную изоморфность** между legacy-ПО и пайплайном анализа данных.

При установленном пороге все реакции в текущем датасете признаны валидными:

RT validity summary: Counter({'OK': 205416})

Использование более жёсткого порога (например, 150 мс) приводит к
искусственному появлению «брака», отсутствующего в исходных данных,
и не рекомендуется.
