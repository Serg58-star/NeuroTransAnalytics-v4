# Рабочее резюме чата: переход к NeuroTransAnalytics-v4

## 1. Статус на момент завершения чата

* Принято окончательное решение **не использовать старый код** как основу для v4.
* Старый проект рассматривается **исключительно как источник данных и понимания их происхождения** (RT, PSI), но **не как архитектурный или методический канон**.
* Виртуальное проектирование v4 (концепции, архитектура, методология) **считается завершённым**.
* Повторный анализ old_docs и старого кода признан **непродуктивным** и прекращён.

## 2. Принятые технические решения по v4 (зафиксированные инварианты)

### 2.1. Среда разработки

* Создан **новый проект**: `NeuroTransAnalytics-v4`.
* IDE: **PyCharm 2025.2**.
* Python: **3.13** (осознанный выбор, без ориентации на старые ограничения).
* GUI-фреймворк: **PyQt6**.

### 2.2. Структура проекта (начальная)

* Проект создан с нуля, без шаблонов.
* Перенесены из предыдущего проекта:

  * каталог `/docs` (каноническая документация v4);
  * каталог `/data` (исторические данные);
  * файлы `.gitignore` и `README.md`.

### 2.3. Git и GitHub

* Инициализирован git-репозиторий.
* Создана ветка разработки: `dev1`.
* Создан и подключён репозиторий **NeuroTransAnalytics-v4** на GitHub.
* Текущее состояние проекта успешно опубликовано.

## 3. Методические фиксации (важно для переноса в будущие чаты)

### 3.1. Статус PSI

* PSI — **контролируемый фактор экспериментального дизайна**.
* PSI — **независимая переменная**, намеренно варьируемая.
* Допустим анализ *влияния PSI на RT*.

**Запрещено**:

* трактовать PSI как измерение;
* трактовать PSI как результат теста;
* использовать PSI как объект QC;
* рассматривать PSI как величину с ошибкой измерения.

### 3.2. Общая позиция по данным

* RT рассматриваются как **сырые измерения**, не подлежащие пост-hoc «улучшению».
* Исторические данные принимаются в том виде, в каком они существуют.
* v4 не обязана воспроизводить аналитическую логику старого проекта.

## 4. Принятая архитектурная рамка v4

* Проект v4 — **новая реализация**, а не рефакторинг.
* Архитектура слоёв (в общем виде):

```
Data layer
Core / Scenarios (C3.4)
Visualization (C3.5)
Interpretation (C4)
GUI (PyQt6, только оркестрация)
```

* Документация в `/docs` считается **первичной**, код — вторичным.

## 5. Решения по работе с ChatGPT

* Планируется создание отдельного проекта ChatGPT: **Testing_RT**.
* Память проекта: **«Только для проекта»**.
* Назначение проекта Testing_RT:

  * сопровождение разработки v4;
  * инженерные решения;
  * работа с данными;
  * генерация и ревью кода;
  * фиксация методических инвариантов.

### 5.1. Что добавляется в проект Testing_RT

* объединённые файлы документации v4 (`.md`);
* 1–2 репрезентативных файла данных (`.xlsx`, `.csv`);
* данный чат — как исторический лог перехода к v4.

### 5.2. Принципы работы в проекте Testing_RT

* не возвращаться к обсуждению старого проекта без явного запроса;
* не повторять концептуальное проектирование;
* работать от зафиксированных решений к реализации;
* фиксировать новые решения как инварианты только после подтверждения.

## 6. Точка выхода из чата

* Контекст чата считается **полностью исчерпанным и зафиксированным**.
* Следующие чаты должны начинаться **не с обсуждений**, а с:

  * проверки документации на противоречия;
  * проектирования адаптеров данных;
  * или непосредственного написания кода v4.
