# 1_Stage G — Архитектура исследовательского генератора v5
## NeuroTransAnalytics — Generator Exploration Stage

Исполнитель: GoAn  
Этап: Stage G (Generator Research Layer)  
Цель: разработка и исследование оптимальной архитектуры генератора стимулов для NeuroTransAnalytics v5.

---

# 1. Назначение Stage G

Stage G предназначен для разработки **исследовательского генератора стимулов**, который позволит:

- системно исследовать влияние структуры теста на компоненты реакции;
- интегрировать эмпирические результаты Stage-L;
- моделировать различные варианты дизайна теста;
- выявлять оптимальные параметры для версии v5.

Stage G является **исследовательским слоем**, а не финальной реализацией генератора.

---

# 2. Основной принцип архитектуры

Генератор должен моделировать **полную цепочку реакции**:

Stimulus Generator
↓
Stimulus Sequence (PSI, позиция, цвет, индекс)
↓
Subject Model (физиология, шум, утомление)
↓
RT
↓
Component Extraction
↓
ΔV1 / ΔV4 / ΔV5/MT


Stage G должен позволять **варьировать параметры на каждом уровне**.

---

# 3. Роль результатов Stage L

Stage-L предоставляет:

- эмпирические распределения ΔV4 и ΔV5/MT;
- медианные значения компонент;
- MAD-разброс;
- зависимости от PSI;
- позиционные эффекты;
- последовательные эффекты.

Эти результаты должны использоваться для:

- калибровки модели субъекта;
- проверки корректности генерации;
- сопоставления симуляций с реальными данными.

---

# 4. Архитектура генератора

Stage G должен состоять из следующих модулей.

---

# Task G-1  
## Модуль генерации стимулов

Создать модуль:

generator/stimulus_generator.py


Функции модуля:

- генерация последовательности стимулов;
- управление PSI;
- управление позициями;
- управление цветами;
- управление порядком стимулов.

Параметры генерации должны быть полностью конфигурируемыми.

---

# Task G-2  
## Модуль PSI как модулятора нагрузки

PSI должен быть реализован как **управляемый модуль нагрузки**.

Для каждой серии должны быть заданы:

- точные значения PSI;
- распределение PSI;
- последовательность PSI.

PSI должен быть:

- предопределён в серии;
- записан в базу данных;
- воспроизводим.

Пример:

psi_series = [800, 1200, 1600, 2000, 2400, 2800]


---

# Task G-3  
## Модуль пространственной геометрии

Создать модуль:

generator/spatial_layout.py


Функции:

- управление позициями стимулов;
- балансировка позиций;
- ограничения повторов.

Поддерживаемые позиции:

L
C
R


---

# Task G-4  
## Модуль цветовых каналов

Создать модуль:

generator/color_channels.py

Каналы:

V1
Parvo
Magno
Koniocellular


Для каждого канала должны быть заданы:

- базовый сдвиг;
- дисперсия;
- параметры шума.

---

# Task G-5  
## Модель субъекта

Создать модуль:

generator/subject_model.py


Модель должна включать:

- базовую скорость реакции;
- индивидуальную вариабельность;
- возраст;
- пол;
- утомление;
- burst-события.

Параметры должны быть настраиваемыми.

---

# Task G-6  
## Модель утомления

Создать модуль:

generator/fatigue_model.py


Функции:

- постепенное смещение медианы;
- рост MAD;
- увеличение вероятности выпадов.

Эффекты должны зависеть от:

- длины серии;
- фазы теста.

---

# Task G-7  
## Модель выпадов (burst events)

Создать модуль:

generator/burst_model.py


Механизм:

- вероятность выпадов;
- распределение величины выброса;
- влияние на RT.

---

# Task G-8  
## Генерация реакции

Создать модуль:

Функция:

RT = f(stimulus, subject_state)


Где учитываются:

- PSI
- канал
- позиция
- состояние субъекта.

---

# Task G-9  
## Извлечение компонент

После генерации RT необходимо вычислять:

ΔV1
ΔV4
ΔV5/MT


Использовать алгоритмы Stage-L.

---

# Task G-10  
## Система конфигурации генератора

Создать файл конфигурации:

generator_config.yaml


В нём должны быть определены:

- структура теста;
- PSI-серии;
- пространственная схема;
- параметры субъекта;
- параметры шума;
- параметры утомления.

---

# Task G-11  
## Запись параметров в базу данных

Каждая серия теста должна фиксироваться в базе данных.

Таблица:

generator_runs

Поля:

run_id
psi_series
stimulus_sequence
generator_config_version
timestamp


Это позволит полностью воспроизводить генерацию.

---

# Task G-12  
## Модуль анализа результатов

Создать модуль:

generator/generator_analysis.py


Он должен сравнивать:

- распределения Δ-компонент;
- медианы;
- MAD;
- PSI-эффекты.

Сравнение проводится между:

Stage L данные
vs
генерация Stage G


---

# 5. Экспериментальные сценарии

Stage G должен поддерживать запуск различных сценариев:

A0 — воспроизведение legacy теста
A1 — варьирование PSI
A2 — варьирование длины серии
A3 — варьирование пространственной структуры
B1 — моделирование утомления
B2 — моделирование выпадов


---

# 6. Критерий успешности Stage G

Stage G считается успешным, если генератор:

- способен воспроизводить распределения Δ-компонент Stage-L;
- позволяет варьировать структуру теста;
- обеспечивает воспроизводимость генерации;
- позволяет экспериментально исследовать параметры теста.

---

# 7. Ожидаемый результат

Stage G должен дать:

- понимание оптимального дизайна теста;
- параметры PSI;
- параметры длины серии;
- параметры геометрии стимулов;
- параметры нагрузки.

Эти результаты будут использованы при финальном проектировании генератора NeuroTransAnalytics v5.
