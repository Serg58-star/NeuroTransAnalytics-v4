# Stage L — Legacy Pattern Extraction Summary

Project: NeuroTransAnalytics-v4
Context: Legacy SZR system forensic analysis
Purpose: Fix the patterns already confirmed in the legacy design and separate them from analytical hypotheses for future verification.

# 1. Цель Stage L

В ходе анализа legacy-системы было сформулировано множество возможных закономерностей.
Однако не все из них подтверждены данными — часть является **следствием структуры системы**, а часть — **гипотезами**, которые необходимо проверять на базе данных.

Задача данного документа:

1. Зафиксировать **фактически подтверждённые свойства legacy-дизайна**.
2. Зафиксировать **аналитические гипотезы**, которые следует проверять на базе данных.
3. Подготовить основу для **проектирования NeuroTransAnalytics v5**.

---

# 2. Источник данных legacy

Важный архитектурный факт:

```text
Legacy database "Boxbase" полностью соответствует базе
neuro_data.db, находящейся в проекте NeuroTransAnalytics-v4.
```

База `neuro_data.db` является **преобразованной копией legacy-базы Boxbase**.

Это означает:

* все анализы legacy-данных могут выполняться **непосредственно на neuro_data.db**
* отдельного доступа к исходной Boxbase не требуется
* структура данных уже адаптирована для анализа в рамках NeuroTransAnalytics.

Таким образом:

```text
Legacy data source for analysis = neuro_data.db
```

# 3. Паттерны, подтверждённые структурой legacy-системы

Эти свойства следуют из анализа:

* stimulus matrix
* config.ini
* reconstructed runtime algorithm
* transition structure

Они **не являются гипотезами** — это реальные свойства системы.

---

## 3.1 Псевдослучайная последовательность PSI

PSI (Pre-Stimulus Interval) распределены псевдослучайно и детерминированы в config.ini.

Назначение:

* разрушение временного предсказания
* предотвращение обучения последовательности
* поддержание состояния постоянной готовности.

---

## 3.2 Псевдослучайная последовательность полей зрения

Стимулы распределены между:

```
left
center
right
```

При этом:

* избегаются длинные серии одного поля
* сохраняется общий баланс.

Это создаёт постоянную необходимость:

```
covert attention shift
```

---

## 3.3 Псевдослучайная последовательность типов стимулов

Типы стимулов:

```
static
motion (shift)
color targets
```

Последовательность построена так, чтобы:

* разрушать предсказание типа события
* создавать переходы между типами обработки.

---

## 3.4 Наличие feature-switch переходов

В stimulus-матрице присутствуют переходы:

```
static → motion
motion → static
color → motion
motion → color
```

Это создаёт нагрузку на системы:

```
perceptual reconfiguration
task switching
```

---

## 3.5 Сбалансированная структура 36 стимулов

Количество стимулов:

```
N = 36
```

Это обеспечивает приблизительный баланс:

```
fields
colors
motion events
```

и их комбинаций.

---

## 3.6 Псевдослучайная система предварительных стимулов

Перед тестом используется **10 вариантов предварительных стимулов**.

Назначение:

* разрушение повторного предсказания
* предотвращение обучения при повторных тестах.

Эти свойства **вытекают из структуры последовательности**, но их проявление должно быть подтверждено анализом базы `neuro_data.db`.

```
high transition entropy
```

## 4.2 Наличие compound transitions

Некоторые переходы включают изменение сразу нескольких параметров:

```
field
feature
motion
```

---

## 4.3 Циклическая структура серии

36 стимулов формируют почти равномерный граф переходов.

# 5. Гипотезы для анализа базы neuro_data.db

Следующие закономерности **требуют проверки**.

---

## 5.1 Feature switch cost

```
RT(static→motion) > RT(motion→motion)
```

---

## 5.2 Spatial transition asymmetry

```
RT(L→R) ≠ RT(R→L)
```

---

## 5.3 Temporal sensitivity

RT может зависеть от PSI-кластера.

---

## 5.4 Peripheral error bias

Ошибки могут чаще возникать в одном поле зрения.

---

## 5.5 Switch-induced errors

Ошибки могут чаще возникать после сложных переходов.

---

## 5.6 Fatigue window

В середине серии может увеличиваться вариабельность RT.
