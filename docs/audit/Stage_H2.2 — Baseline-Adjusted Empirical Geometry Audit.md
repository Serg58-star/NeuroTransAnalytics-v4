# Stage_H2.2 — Baseline-Adjusted Empirical Geometry Audit

## Исполнитель: GoAn

## Роль ChatGPT: последующая строгая методическая верификация

---

# 1. Основание

В ходе Stage_H2 выявлено:

* Эмпирическая геометрия Stage 7 строилась на агрегированных медианах (F1, F2, F3).
* F2 и F3 содержат F1 внутри себя (F2 = F1 + ΔV4; F3 = F1 + ΔV5).
* Это создало искусственную коллинеарность и доминирование PC1.
* Synthetic v5 сравнивалась с эмпирикой в несопоставимом признаковом пространстве (12D vs 3D агрегаты).

Следовательно:

> Требуется пересчёт эмпирической геометрии в корректном базисе признаков.

Без агрегирования.
Без скрытой линейной зависимости.

---

# 2. Цель Stage_H2.2

Провести полный аудит эмпирической геометрии в базисе, который:

* устраняет встроенную зависимость F2 и F3 от F1,
* явно выделяет нагрузочные компоненты (Δ),
* включает латерализацию,
* использует максимально развернутый набор признаков исторической архитектуры.

---

# 3. Формирование корректного признакового пространства

## 3.1 Базовый компонент

* F1 (ПЗР) — baseline

## 3.2 Нагрузочные компоненты

* ΔV4 = F2 − F1
* ΔV5 = F3 − F1

## 3.3 Латерализация (обязательный блок)

Для каждого теста:

* Left field
* Central field
* Right field

Сформировать:

* абсолютные значения по полям,
* Δ по полям,
* латеральные индексы (Left − Right),
* возможно нормированные латеральные коэффициенты.

## 3.4 Минимальные требования

Нельзя ограничиваться только:

(F1, ΔV4, ΔV5)

Нужно построить:

**Full Non-Aggregated Feature Matrix**, включающую:

* каналы,
* поля зрения,
* нагрузочные переходы,
* латерализацию.

Без включения κ, Severity, synthetic и v5.
