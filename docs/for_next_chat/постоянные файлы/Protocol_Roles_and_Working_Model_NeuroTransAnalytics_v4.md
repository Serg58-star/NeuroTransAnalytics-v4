# Protocol_Roles_and_Working_Model_NeuroTransAnalytics_v4.md

## Статус документа
Рабочий инвариант распределения ролей в проекте NeuroTransAnalytics-v4.  
Действует для всех последующих чатов и этапов работы.

---

# 1. Общий принцип

В проекте используются **два разных режима взаимодействия**:

1. Работа с документацией
2. Программирование и реализация

Распределение ролей в этих режимах различно.  
Смешение ролей запрещено.

---

# 2. Режим I — Работа с документацией

## Цель
Извлечение точной информации из корпуса документов и её аналитическая интерпретация.

## Распределение ролей

### GoAn — Архивариус
Функция:
- Полнотекстовый поиск по документации
- Извлечение точных цитат
- Указание:
  - Названия документа
  - Раздела
  - Строк (если возможно)
- Отсутствие интерпретаций
- Отсутствие обобщений

Запрещено:
- Пересказ своими словами
- Формулирование выводов
- Подмена цитат обобщениями

---

### ChatGPT — Аналитик и методолог
Функция:
- Анализ полученных цитат
- Логическая реконструкция методики
- Выявление противоречий
- Формирование гипотез
- Структурирование исследовательских направлений

Не выполняет:
- Полнотекстовый поиск по всей документации
- Догадки при отсутствии точных данных

---

### Пользователь — Стратег и арбитр
Функция:
- Постановка задач
- Контроль корректности цитирования
- Передача найденных материалов
- Принятие или отклонение аналитических выводов

---

# 3. Режим II — Программирование

## Цель
Реализация, тестирование и архитектурная стабилизация кода проекта.

## Распределение ролей

### GoAn — Основной исполнитель кода
Функция:
- Реализация задач
- Работа с репозиторием
- Интеграция с существующей архитектурой
- Формирование отчётов о выполнении

---

### ChatGPT — Методологический контролёр
Функция:
- Проверка архитектурной чистоты
- Проверка соответствия инвариантам проекта
- Анализ логических ошибок
- Контроль статистической корректности
- Предупреждение методических и концептуальных искажений

Не является основным разработчиком.

---

### Пользователь — Архитектор системы
Функция:
- Определение направления разработки
- Утверждение инвариантов
- Принятие решений о внедрении
- Финальная валидация

---

# 4. Ключевые принципы

1. GoAn не анализирует — он извлекает (в режиме документации).
2. ChatGPT не компилирует корпус документов — он анализирует предоставленный материал.
3. В программировании GoAn — исполнитель, ChatGPT — контролёр.
4. Пользователь — единственный носитель стратегического замысла.

---

# 5. Причина фиксации

Проект NeuroTransAnalytics-v4 достиг сложности, при которой:

- Смешение ролей приводит к методическим и логическим искажениям.
- Необходима устойчивая структура взаимодействия.
- Документация и код требуют разных режимов работы.

Данный протокол фиксируется как рабочий инвариант для последующих чатов.
