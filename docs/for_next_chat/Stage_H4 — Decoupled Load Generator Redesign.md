---

# 1. Основание

Stage_H3.1 установил:

* Baseline (F1) — нейронально-глобальный, 1D-доминантный.
* Демографические эффекты выражены амплитудно.
* Моторный слой не объясняет baseline-коллапс.

Stage_H3.2 установил:

* Δ-геометрия (функциональные переходы) структурно корректно реализована в v5.
* Однако:

  * Масштаб нагрузок занижен (в 2–3 раза).
  * Демографическая зависимость нагрузок ошибочно встроена.
  * Реально Δ — почти полностью индивидуальный state-показатель (>99.9% residual).

Следовательно:

> Архитектурная ошибка v5 заключается в том, что функциональная нагрузка (Δ) детерминируется baseline-демографией.

---

# 2. Цель Stage_H4

Перепроектировать генератор Load-переходов (ΔV4, ΔV5, ΔLat) так, чтобы:

1. Он был **статистически независим** от baseline-демографической структуры.
2. Сохранял эмпирическую Δ-геометрию (Rank ≈ 3.7).
3. Воспроизводил реалистичный абсолютный масштаб нагрузок.
4. Не нарушал фазовую логику v5.

---

# 3. Архитектурные принципы нового Load-генератора

## 3.1 Жёсткое разделение слоёв

Новая модель должна быть:

[
Baseline_i = f_{baseline}(Age, Sex, GlobalFactor)
]

[
LoadPenalty_i = g_{state}(LatentState_i)
]

где:

[
Cov(LoadPenalty_i,\ Baseline_i) \approx 0
]

Запрещается:

[
LoadPenalty_i = h(Baseline_i)
]

---

# 4. Новая формальная структура

## 4.1 Latent State Variable

Ввести независимую латентную переменную:

[
S_i \sim Distribution_{state}
]

Параметры S_i:

* Heavy-tailed
* Право-асимметричное
* Независимо от Age и Sex

## 4.2 Load Transition Model

[
ΔV4_i = a_4 \cdot S_i + \epsilon_{4i}
]

[
ΔV5_i = a_5 \cdot S_i + \epsilon_{5i}
]

[
ΔLat_i = a_{lat} \cdot S_i + \epsilon_{lati}
]

где:

* ε — ортогональные шумовые компоненты
* Cov(ΔV4, ΔV5) ≈ эмпирическая
* Rank ≈ 3.7

* SD ≈ 40–55 мс

* Skewness > 2
