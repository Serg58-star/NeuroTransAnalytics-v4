## 2. Структура ключевых таблиц

На основании вывода `PRAGMA table_info`, ядро данных распределено между двумя основными таблицами.

### 2.1 Таблица `users` (Субъекты и демография)

- `subject_id` (INTEGER) — Primary Key
- `last_name` (TEXT)
- `birth_date` (TEXT)
- `first_test_date` (TEXT)
- `gender` (INTEGER)

### 2.2 Таблица `trials` (Данные тестирования)

- `trial_id` (INTEGER) — Primary Key
- `subject_id` (INTEGER)
- `test_date` (TEXT)
- `test_time` (TEXT)
- `session_condition` (INTEGER)
- Набор полей `tst1_1`...`tst1_36`, `tst1_premature`, `tst1_late`, `tst1_mean`, `tst1_std` (и аналогично для `tst2_*` и `tst3_*`) для хранения реакций.

## 3. Выводы по требованиям 9D.0.1 (Без предположений)

- **Таблица субъектов:** `users`
- **Таблицы тестирования:** `trials`
- **Таблица демографии:** `users` (демографические данные включены в таблицу субъектов)
- **Идентификатор субъекта:** Поле `subject_id` присутствует в обеих таблицах (`users` и `trials`).
- **Наличие поля пола:** Да, присутствует поле `gender` в таблице `users`.
- **Кодировка пола:** Тип данных `INTEGER`. Схема базы не содержит текстового маппинга внутри, поэтому кодировка числовая (дополнительного словаря не обнаружено в `sqlite_master`).
- **Наличие даты тестирования:** Да, `test_date` (TEXT) в таблице `trials` и `first_test_date` (TEXT) в таблице `users`.
- **Возможность вычисления возраста:** Возраст может быть вычислен на момент тестирования как разница между `trials.test_date` и `users.birth_date`. Соответственно, вычисление возраста для каждого конкретного теста **возможно**.
- **Множественные тесты:** Архитектура `one-to-many` подтверждена структурой ключей — один `subject_id` (PK в `users`) может встречаться множество раз в таблице `trials`, где уникальность строкам задает ключ `trial_id`.

## 4. Схема связей (Join-логика)

Обе таблицы `users` и `trials` содержат поле `subject_id`.

```sql
SELECT *
FROM trials
JOIN users ON trials.subject_id = users.subject_id;
```

Для исследования популяционных различий (Stage 9D.2) данные будут агрегироваться/фильтроваться с использованием этого JOIN. Возраст на момент теста будет определяться вычислением разницы (в годах/днях) между строками `trials.test_date` и `users.birth_date`.
