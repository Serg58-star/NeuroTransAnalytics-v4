1. Контекст проекта

Проект NeuroTransAnalytics анализирует массив данных зрительно-моторных реакций, полученных в трёх тестах реакционного комплекса:

Tst1 — простая зрительная реакция

Tst2 — реакция на цвет

Tst3 — реакция на движение

Эти тесты являются частями единой архитектуры зрительной реакции, реализованной в программном комплексе тестирования.
Программная система представляет собой строго детерминированный экспериментальный протокол со стандартными последовательностями стимулов и временных интервалов. 

Result_Processing_Code

Проект не исследует сами значения RT как таковые.
Он исследует архитектуру реакции, состоящую из компонент времени.

2. Правильная архитектура зрительной реакции

В проекте используется следующая архитектурная модель:

ΔV1 → ΔV4 → RT
ΔV1 → ΔV5/MT → RT

где:

ΔV1
— базовая сенсомоторная компонента реакции

ΔV4
— центральная компонента принятия решения при узнавании цвета

ΔV5/MT
— центральная компонента принятия решения при реакции на движение

RT является суммарным временем реакции и включает ΔV1 и одну из центральных компонент. 

6_Test_design_and_PSI_metadata_…

Ключевой принцип архитектуры:

ΔV4 и ΔV5/MT — это не независимые измерения.
Они вычисляются как производные от RT и ΔV1.

3. Критическая ошибка, допущенная в текущем чате

В данном чате анализ Stage L был выполнен в неверной архитектуре данных.

Ошибка заключалась в следующем:

Анализ проводился напрямую на RT

Исследовались:

автокорреляции RT

распределения RT

последовательности RT

статистика RT

то есть фактически анализировались:

Tst1

Tst2

Tst3

как три независимых массива RT.

Это является методологической ошибкой.

4. Почему это ошибка

RT не является целевой исследовательской величиной проекта.

RT — это:

сумма компонент

RT = ΔV1 + ΔV4     (цветовой тест)
RT = ΔV1 + ΔV5/MT  (двигательный тест)

Если анализировать RT напрямую:

компоненты смешиваются

архитектура реакции теряется

интерпретация становится невозможной

Фактически в текущем чате анализировалась суперпозиция компонент, а не сами компоненты.

5. Что должно было анализироваться

В Stage L должны анализироваться центральные компоненты реакции:

ΔV4

(цветовое принятие решения)

ΔV5/MT

(двигательное принятие решения)

Эти величины являются целевыми объектами анализа проекта.

RT используется только как исходный материал для их вычисления.
