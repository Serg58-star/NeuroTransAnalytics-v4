# Component Extraction Specification

## Расчёт компонент ΔV1, ΔV4, ΔV5/MT из legacy данных

Project: **NeuroTransAnalytics v4**
Layer: **C3.2 Component Timing**

---

# 1. Назначение документа

Документ фиксирует **алгоритм извлечения компонент временной архитектуры реакции** из legacy-данных системы тестирования.

Компоненты:

* ΔV1
* ΔV4
* ΔV5/MT

вычисляются **детерминированно из RT**.

---

# 2. Исходные данные

Для каждого стимула доступны:

RT
stimulus_color
stimulus_location
stimulus_index
psi

RT хранится в базе данных:

TstX_1 … TstX_36

где X ∈ {1,2,3}

---

# 3. Типы тестов

## Tst1 — простая реакция

Функция:

измерение **базовой сенсомоторной компоненты**

Используется для оценки:

ΔV1

---

## Tst2 — реакция на цвет

Функция:

определение компоненты

ΔV4

---

## Tst3 — реакция на движение

Функция:

определение компоненты

ΔV5/MT

---

# 4. Расчёт ΔV1

ΔV1 определяется на основе данных Tst1.

Используется:

робастная оценка базовой латентности.

Рекомендуемый алгоритм:

ΔV1 = median(Tst1_valid_RT)

или

ΔV1 = percentile(Tst1_RT, 10)

Цель:

получить **минимально достижимое время сенсомоторной реакции**.

---

# 5. Расчёт ΔV4

Для каждого события Tst2:

ΔV4 = RT_Tst2 − ΔV1

где

RT_Tst2 — зарегистрированное время реакции в цветовом тесте.

---

# 6. Расчёт ΔV5/MT

Для каждого события Tst3:

ΔV5/MT = RT_Tst3 − ΔV1

где

RT_Tst3 — зарегистрированное время реакции в тесте движения.

---

# 7. Ограничения значений

Компоненты должны удовлетворять:

ΔV4 ≥ 0
ΔV5/MT ≥ 0

Отрицательные значения означают:

* шум измерения
* ошибку классификации
* аномальную реакцию

и должны маркироваться:

component_flag = anomaly

---

# 8. Таблица ComponentTiming

Результат вычислений сохраняется в таблице:

ComponentTiming

| field               |
| ------------------- |
| stimulus_event_id   |
| delta_v1            |
| delta_v4            |
| delta_v5_mt         |
| computation_version |

---

# 9. Версионность алгоритма

Каждый расчёт компонент должен иметь:

computation_version

например

component_algo_v1

Это необходимо для воспроизводимости результатов.

---

# 10. Роль компонент

Компоненты используются в анализе:

ΔV1
— базовая ось реакции

ΔV4
— архитектура цветового принятия решения

ΔV5/MT
— архитектура двигательного принятия решения

---

# 11. Принцип анализа

В проекте NeuroTransAnalytics анализируются **компоненты реакции**, а не RT.

RT используется **только как исходная величина** для их вычисления.
