# Stage-L Reconstruction Plan

## Пошаговый план переписывания анализа Stage L

Project: **NeuroTransAnalytics v4**
Status: **Architectural Correction Plan**

---

# 1. Назначение документа

Документ описывает **пошаговый план реконструкции анализа Stage L**, который был выполнен в неправильной архитектуре данных.

В текущем чате анализ проводился непосредственно на:

* RT (reaction time)
* Tst1
* Tst2
* Tst3

Это является **методологически неверным**, поскольку объектом анализа проекта являются **компоненты временной архитектуры реакции**, а не суммарное время реакции.

Stage L должен быть **полностью переориентирован на анализ компонент**:

* ΔV1
* ΔV4
* ΔV5/MT

---

# 2. Правильная архитектура реакции

В проекте используется следующая архитектура:

ΔV1 → ΔV4 → RT
ΔV1 → ΔV5/MT → RT

где

RT = ΔV1 + ΔV4
RT = ΔV1 + ΔV5/MT

Следовательно:

RT **не является целевой исследовательской переменной**.

RT используется только как **источник вычисления компонент**.

---

# 3. Цель реконструкции Stage L

Основная задача:

**перенаправить весь существующий аналитический код
с RT на компонентные переменные.**

То есть:

RT → ComponentTiming

где таблица содержит:

* delta_v1
* delta_v4
* delta_v5_mt

---

# 4. Общая архитектура реконструкции

Реконструкция Stage L состоит из **четырёх этапов**.

---

# 5. Этап 1 — восстановление компонент

## 5.1 Создание слоя ComponentTiming

Создаётся таблица:

ComponentTiming

| field               | description             |
| ------------------- | ----------------------- |
| stimulus_event_id   | ссылка на стимул        |
| delta_v1            | базовая компонента      |
| delta_v4            | цветовая компонента     |
| delta_v5_mt         | двигательная компонента |
| computation_version | версия алгоритма        |

---

## 5.2 Источники данных

Используются события:

StimulusEvent
ResponseEvent

и поле

RT

---

## 5.3 Расчёт компонент

Компоненты рассчитываются:

ΔV1
ΔV4
ΔV5/MT

в соответствии с алгоритмом **Component Extraction Specification**.

---

# 6. Этап 2 — формирование компонентного датасета

После расчёта компонент создаётся таблица:

component_dataset

| column            |
| ----------------- |
| subject_id        |
| session_id        |
| stimulus_index    |
| psi               |
| stimulus_color    |
| stimulus_location |
| delta_v1          |
| delta_v4          |
| delta_v5_mt       |
| error_type        |

---

# 7. Этап 3 — переориентация анализа

Все аналитические процедуры Stage L должны быть переподключены к компонентам.

---

## 7.1 Анализ распределений

Было:

distribution(RT)

Должно быть:

distribution(ΔV4)
distribution(ΔV5/MT)

---

## 7.2 Автокорреляционный анализ

Было:

autocorr(RT)

Должно быть:

autocorr(ΔV4)
autocorr(ΔV5/MT)

---

## 7.3 PSI-анализ

Было:

RT ~ PSI

Должно быть:

ΔV4 ~ PSI
ΔV5/MT ~ PSI

---

## 7.4 Позиционный анализ

Было:

RT ~ stimulus_position

Должно быть:

ΔV4 ~ position
ΔV5/MT ~ position

---

## 7.5 Пространственный анализ

Было:

RT ~ left / center / right

Должно быть:

ΔV4 ~ location
ΔV5/MT ~ location

---

# 8. Этап 4 — популяционные структуры

После перехода на компонентные переменные выполняется:

* кластерный анализ
* поиск редких паттернов
* сравнение популяций

на основе:

ΔV4
ΔV5/MT

---

# 9. Сохранение существующего кода

Большая часть существующего кода может быть сохранена.

Без изменений остаются:

* ETL
* загрузка данных
* обработка PSI
* позиционные индексы
* группировки
* визуализации

Изменяется только **источник переменной анализа**.

---

# 10. Что запрещено в Stage L

Stage L **не должен анализировать напрямую:**

* RT
* Tst1
* Tst2
* Tst3

Эти величины используются **только как исходные данные для вычисления компонент**.

---

# 11. Итог

Stage L после реконструкции становится:

Component-level analysis stage

где анализируются:

ΔV4
ΔV5/MT

а не RT.
