# NeuroTransAnalytics v4

NeuroTransAnalytics — исследовательский программный проект на Python,
предназначенный для анализа скоростей зрительных реакций (СЗР)
в индивидуальном и популяционном разрезе
на основе исторического, строго стандартизированного массива данных.

Проект реализуется как **исследовательская аналитическая система**,  
а не как диагностический или клинический инструмент.

---

# 1. Источник данных (КЛЮЧЕВО)

Единственным источником данных проекта является база данных:

**SQLite: `neuro_data.db`**

Все расчёты, сценарии анализа, визуализации и интерпретации
выполняются **исключительно** на данных, содержащихся в этой базе.

Исходные файлы форматов:

- xlsx
- ini
- mdb

использовались **только на этапе исторической миграции**
и **не входят в рабочий контур проекта v4**.

# 2. Состав базы данных `neuro_data.db`

## 2.1 Таблица участников

- `users` — демографическая информация испытуемых  
  (идентификатор, пол, дата рождения, дата первого тестирования)

## 2.2 Таблица тестовых сессий и реакций

- `trials` — результаты всех тестовых сессий:
  - временные параметры,
  - 36 реакций для каждого теста (Tst1, Tst2, Tst3),
  - показатели ошибок и агрегатов, сохранённые в историческом виде.

## 2.3 Метаданные тестовых стимулов

- `metadata_simple` — 36 предъявлений теста простой реакции (Tst1)
- `metadata_color_red` — 36 предъявлений теста на красный цвет (Tst2)
- `metadata_shift` — 36 предъявлений теста со сдвигом (Tst3)

Метаданные содержат:

- цвет стимула
- пространственную локализацию
- предстимульный интервал (PSI)
- параметры маскировки и сдвига

## 2.4 Разминочные стимулы

- `warmup_simple`
- `warmup_color_red`
- `warmup_shift`

Разминочные стимулы **не являются строками анализа**,  
но участвуют в формировании PSI и контекста реакции.

## 2.5 Системные параметры

- `system_parameters` — константы экспериментальной установки

# 3. Архитектура анализа

Проект v4 организован по уровням:

- **A** — архитектура временных компонент реакции (ΔV1, ΔV4, ΔV5)
- **B** — условия тестирования как инструментальные зонды (PSI, позиция, маскировка)
- **C** — проектирование, хранение, вычисления, визуализация и интерпретация

GUI никогда не выполняет вычислений.

---

# 4. Методологические ограничения

Проект **не выполняет**:

- клиническую диагностику
- физиологическую локализацию
- автоматическую классификацию состояний
- причинные интерпретации

Все выводы носят:

- описательный
- архитектурный
- статистически-обусловленный характер

# 5. Статус математической валидации (v4 Core)

В рамках Exploratory Architecture Framework выполнены:

- Stage 1–6 — Structural Verification  
- Stage 7 — Population Geometry (Continuum confirmed)  
- Stage 8 — Trait vs State Dynamics (Dynamically Structured Manifold)  
- Stage 8.5 — Mathematical Norm Envelope (Stable Norm Core)

## Подтверждённые инварианты:

- Латентное пространство 3D.
- Пространство непрерывное (нет дискретных типов).
- Геометрия популяционно инвариантна.
- Динамика марковская (H ≈ 0.5).
- Норма формализована через MCD и Mahalanobis Distance.
- Границы нормы устойчивы к шуму.

C3-Core признан математически валидированным.

Проект готов к Stage 9 — Clinical / External Validation.

---

# 6. Назначение проекта

NeuroTransAnalytics предназначен для:

- извлечения закономерностей из уникального legacy-массива данных
- проверки устойчивости архитектур зрительной реакции
- проектирования следующих поколений тестов СЗР
- количественной оценки отклонений от математической нормы

Проект сознательно сохраняет сложность данных
и не оптимизируется под «усреднённого испытуемого».
